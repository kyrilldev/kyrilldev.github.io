Code HighLights and Explanations

The 'TiltCamera' method manages the tilting behavior of the in-game camera in response to player movement. Key features of this code include:

  • Crouch Check: Ensures that the camera doesn't tilt when the player is crouching, maintaining stability during crouched movement.
  • Dynamic Tilt Calculation: Utilizes player input ('playerManager.playerMovement.x') to calculate the desired tilt angle ('tilt'). The tilt is determined based on whether the player is moving left, right, or not moving at all.
  • Cross-Platform Input Handling: Implements a switch statement with comparison operators ('<'', '>') instead of explicit values ('== 1' or '== -1') to accommodate various input methods, ensuring cross-platform compatibility.
  • Tilt Smoothness: Applies a smoothing effect to the calculated tilt using 'Mathf.Lerp' to gradually adjust the 'targetMoveTilt'. The tilt is influenced by the player's vertical movement ('playerManager.playerMovement.y'), resulting in a different tilt when moving forward or backward.


    • The Fire method manages the player's firing action, handling the instantiation and initialization of bullets. Key features of this code include:

  • Input Action Callback: Responds to the firing action callback provided by the 'InputAction.CallbackContext' parameter. This ensures that the firing logic is triggered only when the firing action starts ('cxt.started').
  • Base Rotation and Position: Captures the initial rotation ('baseRotation') and position ('basePosition') of the 'Shootpoint', which is likely the point from which bullets are fired.
  • Object Pooling: Utilizes an object pool ('Pool.GetPooledObject') to efficiently retrieve a bullet object from the pool, considering the initial rotation and position.
  • Bullet Component Addition: Dynamically adds a 'Bullet' component to the instantiated bullet object if it doesn't already have one. This ensures consistent behavior and properties for all bullets.
  • Bullet Direction Calculation: Determines the shooting direction ('shootDir') based on the vector pointing from the player's position to the 'Shootpoint'. This normalized direction provides the initial trajectory of the fired bullet.
  • Bullet Initialization: Calls the 'Init' method on the 'Bullet' component of the instantiated object, passing the calculated shooting direction to initialize the bullet's behavior.


    • The 'Headbob' method orchestrates a dynamic headbobbing effect for the player's camera, enhancing realism and immersion. Key features of this code include:

  • Reduce Motion Handling: Checks if the player has opted to reduce motion effects, resetting relevant vectors to zero and bypassing the headbobbing calculations if enabled. This accommodates players who may prefer a motion-reduced experience.
  • Fall-induced Bobbing: Triggers a brief bobbing effect when the player falls and then lands, adding a subtle visual cue to the impact. This utilizes the 'BobOnce' method.
  • Crouch and Sprint Bobbing: Adjusts the headbobbing frequency based on whether the player is crouching and sprinting simultaneously, enhancing the effect during specific movement conditions.
  • Frequency Adjustment: Calls the 'desiredFreq' method to dynamically adjust the headbobbing frequency based on movement conditions. The frequency adjustment is performed smoothly using 'Vector3.Lerp'.
  • Land Bobbing Offset: Utilizes lerping to gradually reduce the 'landBobOffset' vector, contributing to a smoother transition between headbobbing states.
  • Motion Check: Calls the 'CheckMotion' method, likely responsible for managing the overall motion effects.


    • The 'Expand' method dynamically expands an object pool by instantiating and initializing a specified number of objects.

    Key features of this code include:

  • Looped Instantiation: Utilizes a 'for' loop to instantiate a set number of objects ('l_ExpandSize') based on the provided parameter.
  • Object Instantiation: Instantiates a new object ('newObject') of the type specified by the 'PooledObject' template.
  • Component Retrieval: Retrieves the 'PoolItem' and 'Subject' components from the instantiated object, allowing for subsequent customization and management.
  • Initialization: Sets the 'PoolProperty' of the 'PoolItem' to link it to the current object pool. Additionally, adds the instantiated object as a subject to the player's health system ('playerHealth._subjects').
  • Global Subjects Assignment: Iterates through an array of enemy health systems ('enemyHealth') and adds the instantiated object as a subject to each enemy health system.
  • Object Return to Pool: Calls the 'ReturnPooledObject' method, likely responsible for returning the instantiated object to the object pool for reuse.


    • The following methods are crucial components of an object pooling system in a Unity game script, written in C#:

    Activate:

  • Purpose: Resets the death timer to zero, effectively reactivating an object.
  • Description: The 'Activate' method is responsible for resetting the 'DeathTimer' variable to zero, indicating that the object is being reactivated. This method is often called when an object is retrieved from the pool for reuse.

    • ReturnToPool:
  • Purpose: Deactivates the object, returning it to the object pool for potential reuse.
  • Description: The 'ReturnToPool' method serves as a centralized control point for deactivating an object and returning it to the object pool ('MyPool'). It calls the DeActivate method to perform necessary deactivation procedures before returning the object to the pool using 'MyPool.ReturnPooledObject(this)'.

    • OnCollisionEnter:
  • Purpose: Triggers object deactivation when a collision is detected.
  • Description: The OnCollisionEnter method is an event handler that automatically invokes the ReturnToPool method when a collision occurs. This design ensures that objects are returned to the pool upon collision, promoting efficient memory management.

In this contract we had to write down in detail what kind of features we wanted to make in the game. I chose to make a shader and a pretty advanced character controller. In the first week I built the map and started working on the character controller. I finished the character controller in the 3rd week and worked to implement the rest of the game and the shader with the time I had left. I ended up making a glowing shader using the language GLSL. After the project was done one of the teachers came by every student to give feedback on the code and the game itself. I enjoyed workin on my own for once and had a blast writing the shader.
