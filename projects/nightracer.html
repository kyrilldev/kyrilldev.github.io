Code HighLights and Explanations

This script checks if the game can spawn puzzle pieces ('_canSpawn' is false) and if the first puzzle piece in the array ('puzzlePieces[0]' ) is not null. If these conditions are met, it uses 'Physics.OverlapSphere' to find colliders within a specified radius around the first puzzle piece's position, using a specified layer mask. The colliders list is then cleared.

It checks if the number of colliders in the colliders list is equal to a specified piece count ('_pieceCount') and if the number of puzzle pieces on the outline is equal to or greater than 4 ('puzzlePieceCount >= 4'). If both conditions are met, it starts a coroutine ('DestroyPuzzlePieces >= 4') to destroy the puzzle pieces.


In this snippet, I've crafted a function named ('AssignWaypoints') designed for a Unity Cinemachine implementation.
This function dynamically assigns positions to a Cinemachine Smooth Path based on the provided waypoints. It also toggles between multiple virtual cameras, enabling the camera associated with the specified level index.
The code not only sets the waypoints for smooth path traversal but also configures the camera to follow and look at a specific target, enhancing the cinematic experience within the game.
This snippet showcases my ability to manage camera systems and create dynamic, immersive experiences in Unity game development.


In this coroutine, aptly named 'PathPosLerp', I implement smooth interpolation of a Cinemachine Virtual Camera along a tracked dolly path in Unity. This function takes parameters such as the virtual camera ('vcam'), the duration of the interpolation ('sec'), and the level index ('l_levelIndex').
During the interpolation, the camera smoothly moves from the starting position to the end position. The code also includes functionality to disable player control during cutscenes by updating a flag in the GameManager.
Finally, the associated virtual camera is disabled upon completion, showcasing my proficiency in creating seamless transitions and enhancing player experiences in Unity game development.


This function, 'OnGUI', defines a custom Unity editor window for convenient configuration of waypoints in the scene. The window provides an intuitive interface for managing waypoints within the game environment. Key features include:

  • Information Label: Describes the purpose of the window, emphasizing its role in configuring waypoints.

  • Camera Manager Setup: Allows for the selection of a 'CameraManager' object through a field. This manager is then located in the scene for further use.

  • Waypoint Configuration: Provides toggles for individual cells, allowing quick and visual configuration of different waypoints in the scene.

  • Spawn Empties Button: Triggers the spawning of entities (empties) based on the configured waypoints. Once spawned, the button sets the level position to zero.

  • Reset Waypoint Positions Button: Initiates a confirmation dialogue before resetting all waypoint positions. Upon confirmation, it resets the positions of associated GameObjects.

    The Assignment

    In the making of this game we were supposed to think of 3 puzzles and we had to combine the 3 puzzle into a smooth flowing game.
    The name of the assignment was "Shortburn", we worked together in groups of 5-6 people. In my case there were 3 developers and 3 artists.
    In the first puzzle as you can see in the video you need to find all the puzzle pieces in the room and stick them on a board to spawn the whole shield. The second room you had to switch pov's to the second smaller player and go through a hole in the wall and through a bunch of parkour you have to make your way to a ball and drop it on a lever that opened the third room.
    In the third room you had to complete a Indiana Jones inspired tile puzzle.
    You could reveal the correct path by throwing rocks on the tiles. When you finished this puzzle you could go to the end door and finish the game.
