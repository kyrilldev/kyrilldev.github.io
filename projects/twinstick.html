Code HighLights and Explanations

The AttackPlayer function handles the logic for an enemy character to attack a player. Key features of this function include:

  • Navigation Reset: Uses Unity's 'NavMeshAgent' to set the destination to the enemy's current position, effectively pausing its movement.
  • Rotation towards Player: Utilizes 'transform.LookAt(player)' to make the enemy character face the player.
  • Projectile Instantiation: Instantiates a projectile ('projectile') at the enemy's position using 'Instantiate', providing it with an initial velocity in the forward direction.
  • Projectile Movement: Translates the enemy character in the forward direction at a specified speed ('projectileSpeed * Time.deltaTime'), simulating the projectile launch.
  • Attack Cooldown: Implements a cooldown mechanism ('timeBetweenAttacks') to prevent the enemy from attacking continuously. The 'Invoke' function is used to schedule the 'ResetAttack' method after a specified delay.
  • Attack Reset: The 'ResetAttack' method resets the 'alreadyAttacked' boolean, allowing the enemy to attack again after the cooldown period.


    • The SetNextTarget function manages the navigation system for an agent, determining its next target waypoint. Key features of this function include:

  • Previous Index Tracking: Stores the current index as 'prevIndex' to keep track of the agent's previous waypoint.
  • Conditional Random Movement: If '_MoveRandom' is enabled, randomly selects a waypoint index from the list ('_Waypoints'). If the new index matches the previous one, increments it to ensure a change in destination.
  • Sequential Movement: If '_MoveRandom' is not enabled, increments the current index in a sequential manner, ensuring the agent moves through waypoints in order.
  • Index Wraparound: Handles index wraparound by using modulo ('_currentIndex %= _Waypoints.Count'), ensuring the agent cycles through waypoints seamlessly.
  • Navigation Destination: Sets the agent's destination to the position of the waypoint corresponding to the calculated index using 'SetDestination'.


    • The Patrolling function governs the patrolling behavior of an agent, ensuring it moves between designated walk points. Key features of this function include:

  • Walk Point Initialization: Checks if a walk point has been set ('!walkPointSet'). If not, it calls 'SearchWalkPoint()' to find a suitable location within the patrol area.
  • Destination Setting: If a walk point is set, uses 'agent.SetDestination(walkPoint)' to guide the agent towards the designated walk point.
  • Distance Check: Calculates the distance between the agent's current position and the walk point using 'Vector3 distanceToWaypoint = transform.position - walkPoint;'.
  • Walk Point Reset: If the agent is close enough to the walk point ('distanceToWaypoint.magnitude < 1f'), it marks 'walkPointSet' as false, indicating that the agent has reached its destination and is ready to search for a new walk point.


    • The Update function governs the behavior of a weapon system, managing ammunition, reloading, and firing. Key features of this function include:

  • Ammo Pickup Handling: Checks if the player has picked up ammo ('AmmoPickUp.hasPickedUpAmmo'). If true, increases the current ammo count by the specified value ('ammoValue') and resets the pickup flag.
  • Ammo Limitation: Ensures that the current ammo does not exceed the defined maximum ammo ('maxAmmo').
  • Reloading Logic: If the player is not already in the process of reloading ('isReloading' is false) and the current ammo is depleted, triggers the 'Reload' coroutine.
  • Fire Input Handling: Listens for input (e.g., left mouse button) and a cooldown mechanism ('Time.time >= nextTimeToFiring'). If conditions are met, initiates the 'Shoot' method and sets the next time to fire based on the defined fire rate.
